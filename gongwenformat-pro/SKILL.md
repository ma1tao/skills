# 党政机关公文标准排版 Skill

依据《党政机关公文格式》国家标准（GB/T 9704-2012），将任意格式文本自动排版为标准公文格式的 Word 文档。

## 何时使用

- 用户写好了研讨材料、发言稿、汇报材料、通知、请示等，需要生成正式Word文件
- 用户要求"排版"、"标准格式"、"公文格式"
- 用户说"发给我Word"、"生成docx"
- 用户发来 TXT、不标准的Word文件，要求排成标准格式

## 排版标准参数

### 页面设置
| 项目 | 数值 |
|------|------|
| 纸张 | A4 (210mm × 297mm) |
| 上边距 | 37mm |
| 下边距 | 35mm |
| 左边距 | 28mm |
| 右边距 | 26mm |
| 版心尺寸 | 156mm × 225mm |
| 每页行数 | 22行 |
| 每行字数 | 28字 |

### 字体字号
| 要素 | 字体 | 字号 |
|------|------|------|
| 标题 | 方正小标宋简体（或SimSun） | 二号 |
| 一级标题（一、二、） | 黑体 | 三号 |
| 二级标题（（一）（二）） | 楷体 | 三号 |
| 三级标题（1. 2.） | 仿宋 | 三号 |
| 四级标题（（1）（2）） | 仿宋 | 三号 |
| 正文 | 仿宋 | 三号 |
| 页码 | 宋体 | 四号半角 |

### 段落格式
- 正文行距：固定值 28磅
- 标题行距：固定值 32磅（标题较长时可适当调整）
- 正文首行缩进：2字符
- 正文中数字、英文使用 Times New Roman 字体
- 标题居中排列，回行时词意完整

### 页码
- 格式：`-1-`，`-2-`（数字左右各一条一字线）
- 单页码居右空一字，双页码居左空一字
- 距版心下边缘 7mm

### 层次序号规范
```
第一层：一、二、三、（黑体）
第二层：（一）（二）（三）（楷体）
第三层：1. 2. 3.（仿宋）
第四层：（1）（2）（3）（仿宋）
```

## 使用方法

### 第一步：确认内容并转为中间文件

从飞书云文档获取内容，或接收用户发来的文件（TXT/Word/直接文字），保存为中间文件。

**输入格式支持：**
- **Markdown (.md)**：自动识别 `##` 标题和 `**加粗**` 标记
- **纯文本 (.txt)**：按空行分段，智能识别层级序号
- **Word (.docx)**：提取纯文本后智能识别层级
- **直接文字**：用户在聊天中发的文字，保存为 .txt 后处理

### 第二步：生成Word文件

执行脚本生成排版好的Word文档：

```bash
python3 <skill-path>/scripts/gongwen_format.py \
  --title "公文标题" \
  --input /tmp/content.md \
  --output /tmp/output.docx
```

参数说明：
- `--title`：公文标题（二号小标宋，居中）
- `--input`：输入文件路径（支持 .md / .txt / .docx）
- `--output`：输出的Word文件路径
- `--author`（可选）：发文机关名称，用于落款
- `--date`（可选）：成文日期，默认为当天

### 第三步：发送文件

将生成的 `.docx` 文件发送给用户。

### 高级参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--author` | 发文机关名称（落款右对齐） | `--author "XX镇人民政府"` |
| `--date` | 成文日期（自动格式化） | `--date "2026-04-16"` 或 `--date "2026年4月16日"` |
| `--print-author` | 印发机关（底部居中） | `--print-author "XX镇人民政府办公室"` |
| `--print-date` | 印发日期（底部居中） | `--print-date "2026-04-17"` |

**日期格式化**：支持 `2026-04-16`、`2026/04/16`、`2026年4月16日` 等格式，自动转为标准中文日期。

**附件处理**：正文中包含 `附件：xxx` 或 `附件:xxx` 时，自动识别为附件说明，使用仿宋三号居左排版。

**页码**：单页码居右空一字、双页码居左空一字，格式 `-1-`，符合国标要求。

## 注意事项

1. **字体兼容**：Linux 环境可能没有方正小标宋简体，脚本会自动降级到 SimSun（宋体），标题使用宋体加粗作为替代
2. **标题分行**：如果标题超过一行，脚本会自动换行并保持词意完整
3. **智能层级识别**（不依赖Markdown语法也能识别）：
   - `一、` `二、` `第X部分` → 一级标题（黑体）
   - `（一）` `（二）` → 二级标题（楷体）
   - `1.` `2.` → 三级标题（仿宋）
   - `（1）` `（2）` → 四级标题（仿宋）
   - Markdown `##` 和 `**加粗**` 也能识别
4. **首行缩进**：正文段落自动首行缩进2字符，标题和列表项不缩进
5. **数字字体**：正文中的阿拉伯数字自动使用 Times New Roman
6. **页码设置**：单页码居右空一字、双页码居左空一字，格式 `-1-`，符合国标
7. **附件标记**：正文中 `附件：标题` 会自动识别并排版
8. **日期格式化**：`--date` 和 `--print-date` 支持多种输入格式，自动转为标准中文日期
9. **印发信息**：`--print-author` 和 `--print-date` 在落款下方添加印发机关和日期
